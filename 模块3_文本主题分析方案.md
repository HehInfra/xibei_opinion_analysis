# 模块 3：文本主题分析方案

## 1. 模块目标

模块 3 负责回答：

```text
西贝事件相关评论中，用户主要在讨论什么？
```

本模块不直接判断用户支持谁、反对谁，也不直接判断情绪好坏，而是将评论文本归入若干事件相关主题。

模块 3 的分析结果将用于：

```text
1. 展示评论区讨论焦点
2. 比较不同视频评论区的主题差异
3. 支撑模块 4 的情绪与立场分析
4. 支撑模块 6 的评论区圈层化分析
```

## 2. 基本判断

当前数据量为 455 条评论/回复，数据规模不大，但事件语境强、评论口语化明显、调侃和反讽较多。

因此，本项目不采用训练专用主题分类模型的方式，而采用：

```text
规则初标 + 可解释原因 + 人工校验 + 必要时大模型辅助
```

原因：

```text
1. 数据量不足以支撑可靠训练
2. 课程设计更需要可解释性
3. 西贝事件主题具有明显关键词和语义线索
4. B 站评论中存在大量梗、调侃和反讽，完全依赖通用模型容易误判
```

## 3. 主题标签体系

```text
每条评论优先归入一个最主要主题
```

必要时可以保留辅助主题，但第一版只使用主主题，降低分析复杂度。

## 4. 主题判定优先级

当一条评论命中多个主题时，按以下优先级判断主主题：

```text
1. 食品安全与儿童餐
2. 消费者知情权
3. 品牌公关与回应翻车
4. 预制菜与食材真实性
5. 价格与性价比
6. 罗永浩个人与较真形象
7. 餐饮行业共性讨论
8. 调侃玩梗
9. 其他 / 不明确
```

优先级理由：

```text
食品安全、消费者权益和品牌回应更接近舆情核心争议；
预制菜和价格是主要事实争议；
罗永浩个人和行业讨论是扩展讨论；
调侃玩梗作为传播形态，但不优先覆盖实质主题。
```

## 5. 分析方法

### 5.1 规则初标

对 `contents.content_text` 进行关键词匹配和简单模式识别。

输出：

```text
topic_label
topic_confidence
topic_reason
topic_matched_keywords
need_review
```

置信度规则：

```text
0.90：命中高优先级主题的多个关键词，且语义明确
0.75：命中一个强关键词或多个弱关键词
0.60：只命中弱关键词，或存在多个主题竞争
0.40：未命中明确关键词，进入 other_unclear
```

### 5.2 人工校验

必须人工校验：

```text
1. 高赞评论 Top 50
2. topic_confidence < 0.70 的评论
3. 被标记为 other_unclear 的评论
4. 同时命中多个强主题的评论
```

人工校验字段：

```text
reviewed_topic_label
review_note
reviewer
reviewed_at
```

### 5.3 大模型辅助

如果时间允许，可使用大模型辅助判断复杂评论。

大模型只作为辅助，不作为唯一依据。

适合大模型处理：

```text
反讽评论
玩梗评论
关键词不明显但语义明确的评论
多主题混合评论
```

大模型输出需要保留解释：

```text
llm_topic_label
llm_reason
```

## 7. 质量控制

模块 3 需要保留以下质量指标：

```text
主题覆盖率
other_unclear 占比
need_review 占比
高赞评论人工校验完成率
低置信度评论人工校验完成率
```

如果 `other_unclear` 占比过高，说明标签体系或关键词规则需要修正。

如果某个主题过度集中，需要检查是否关键词过宽。

## 8. 与后续模块的关系

模块 3 输出的 `topic_label` 将被模块 6 使用，用于计算：

```text
主题集中度
同主题互动比例
跨主题互动比例
视频评论区主题多样性
```

模块 3 不直接判断情绪和立场，但它为模块 4 提供上下文。

例如：

```text
同样是负面评论，
如果主题是 price_value，可能表示价格不满；
如果主题是 brand_pr_crisis，可能表示对品牌回应不满。
```
