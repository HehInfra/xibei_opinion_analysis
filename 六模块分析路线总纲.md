# 六模块分析路线总纲

## 1. 总体目标

本项目围绕西贝事件相关社交媒体数据，采用六个分析模块逐层推进。

整体思路是：

```text
先还原数据事实
再识别热度与注意力
再分析文本主题
再判断情绪与立场
再构建互动网络
最后综合判断评论区圈层化与事件型信息茧房倾向
```

六个模块不是彼此孤立的统计任务，而是一条从原始事实到解释结论的分析链。

```text
模块 1：数据概览
    ↓
模块 2：热度分析
    ↓
模块 3：文本主题分析
    ↓
模块 4：情绪与立场分析
    ↓
模块 5：用户互动网络分析
    ↓
模块 6：评论区圈层化分析
```

## 2. 分析对象

当前主要数据来源为 B 站视频评论数据。

第一层事实数据已经整理为：

```text
sources：视频源
users：评论用户
contents：一级评论与楼中楼回复
relations：评论视频关系与用户回复关系
```

后续分析均基于第一层事实数据展开。

需要注意：

```text
当前数据更适合分析“视频评论区中的舆论聚集和互动结构”，
不应直接宣称已经覆盖完整社交平台传播网络。
```

因此，项目表述应更准确地定位为：

```text
西贝事件在 B 站视频评论区中的观点聚集、情绪表达、互动关系与局部信息茧房现象分析。
```

如果后续补充微博、抖音、小红书或新闻评论数据，可以再扩展为跨平台舆情分析。

## 7. 模块 5：用户互动网络分析

### 7.1 核心问题

模块 5 回答：

```text
谁和谁发生了互动？
评论区是否存在中心用户？
楼中楼回复是否形成局部讨论圈？
不同立场用户之间是否发生互动？
```

### 7.2 网络类型

当前数据支持两类网络：

```text
强关系网络：用户 A 回复 用户 B
弱关系网络：用户 A 和 用户 B 共同评论同一个视频
```

第一版优先分析强关系网络：

```text
reply 网络
```

因为它是真实用户对用户互动。

第二版可以构造弱关系：

```text
co_participation 网络
```

但弱关系需要明确标注为分析构造关系，而不是平台原始互动关系。

### 7.3 输入数据

```text
relations
users
contents
模块 3 topic_label
模块 4 sentiment_label / stance_label
```

### 7.4 主要分析内容

```text
节点数量
边数量
网络密度
入度/出度
被回复最多用户
主动回复最多用户
弱连通分量
立场同质性
跨立场回复比例
```

## 8. 模块 6：评论区圈层化分析

### 8.1 核心问题

模块 6 回答：

```text
评论区是否存在观点聚集、情绪同质和互动封闭？
是否表现出事件型信息茧房或局部回音室倾向？
```

### 8.2 分析对象

模块 6 不直接分析全网传播，而分析：

```text
B 站视频评论区中的局部圈层化现象
```

### 8.3 输入数据

```text
模块 3 topic_label
模块 4 sentiment_label / stance_label
模块 5 reply 网络
模块 2 高赞评论和热度结果
```

### 8.4 指标设计

评论区圈层化可以由四类指标构成：

```text
主题集中度
情绪一致性
立场一致性
互动同质性
```

可设计为：

```text
CocoonScore =
  0.25 * TopicConcentration
+ 0.25 * SentimentConsistency
+ 0.30 * StanceConsistency
+ 0.20 * InteractionHomophily
```

其中：

```text
TopicConcentration：视频内最大主题占比
SentimentConsistency：视频内最大情绪占比
StanceConsistency：视频内最大立场占比
InteractionHomophily：回复关系中同立场互动比例
```

如果某视频回复关系太少，则：

```text
InteractionHomophily 标记为样本不足
CocoonScore 可以给出文本版分数和结构版分数两种
```

### 8.5 预期输出

```text
data/analysis/module6_cocoon/cocoon_overview.json
data/analysis/module6_cocoon/source_cocoon_scores.csv
data/analysis/module6_cocoon/topic_concentration.csv
data/analysis/module6_cocoon/sentiment_consistency.csv
data/analysis/module6_cocoon/stance_consistency.csv
data/analysis/module6_cocoon/interaction_homophily.csv
data/analysis/module6_cocoon/summary.md
```

### 8.6 模块作用

模块 6 是最终解释层。

它将回答：

```text
哪些视频评论区更容易形成单一观点聚集？
哪些评论区以讽刺/负面情绪为主？
高赞评论是否强化了某种立场？
用户回复是否主要发生在相同立场之间？
```

## 9. 六模块之间的数据依赖

```text
layer1 数据
    ↓
模块 1：数据概览
    ↓
模块 2：热度分析
    ↓
模块 3：主题标签
    ↓
模块 4：情绪与立场标签
    ↓
模块 5：互动网络
    ↓
模块 6：圈层化指标
```

更具体地说：

```text
模块 3 依赖 contents
模块 4 依赖 contents + 模块 3
模块 5 依赖 relations + 模块 3 + 模块 4
模块 6 依赖 模块 2 + 模块 3 + 模块 4 + 模块 5
```

## 10. 当前已确定的方法边界

当前阶段采用：

```text
规则初标
+ DeepSeek 大模型校正
+ 暂时跳过人工校验
```

但是报告中必须说明：

```text
人工校验尚未执行
need_review 字段用于标记后续需要复核的样本
当前标签结果适合原型分析和趋势判断
不应作为高精度人工标注数据集使用
```

## 11. 最终汇报建议

最终展示时，可以按以下逻辑讲：

```text
1. 我们先构建第一层事实数据：视频、用户、评论、关系
2. 再通过模块 1 和 2 了解数据规模与热度结构
3. 通过模块 3 识别评论区讨论主题
4. 通过模块 4 判断情绪和立场
5. 通过模块 5 构建用户互动网络
6. 通过模块 6 综合评估事件型信息茧房倾向
```

这条路线体现了：

```text
数据工程
文本理解
关系网络
舆情解释
可视化展示
```

四者之间的连续关系。
