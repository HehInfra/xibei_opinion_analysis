# 02 语义分析与模型微调：从标签体系到全量预测

## 1. 为什么不只做情绪分类

如果只把评论分成正向、中性、负向，西贝事件中最重要的结构会被压扁。

例如，同样是负面或质疑表达，评论可能在讨论：

1. 预制菜和食材真实性。
2. 价格和性价比。
3. 品牌公关回应翻车。
4. 罗永浩个人形象和维权叙事。
5. 消费者知情权。
6. 玩梗、反讽和跑题传播。

这些讨论对象不同，内容安全含义也不同。质疑西贝不一定是风险；反讽和玩梗不一定是攻击；高赞负面评论也可能是对品牌回应的合理批评。因此项目后续从“情绪分析”扩展为“多维评论语义分析”。

## 2. 标签体系设计

项目设计了六类语义维度：

| 维度 | 类型 | 含义 |
|---|---|---|
| topic | 单标签 | 评论主要讨论什么 |
| stance_target | 单标签 | 评论主要针对谁 |
| stance_label | 单标签 | 评论是支持、质疑、中立还是不明确 |
| emotion_label | 单标签 | 评论表达什么情绪 |
| discourse_labels | 多标签 | 评论如何表达，例如反讽、质问、玩梗 |
| risk_labels | 多标签 | 评论是否包含内容安全风险表达 |

本轮实际完成训练和全量预测的是前四个单标签维度：

```text
topic
emotion
stance_target
stance
```

话语方式和风险特征已经有伪标签训练数据，但尚未正式训练多标签分类器。

## 3. 标注与训练集构建

项目先从 40803 条全量评论中抽样 1500 条，用于语义标注。抽样策略不是完全随机，而是结合：

1. 高赞评论。
2. 高回复评论。
3. 视频源分层抽样。
4. 全局随机抽样。

这样做是为了让训练样本既覆盖普通评论，也覆盖更容易影响舆论可见性的高互动评论。

随后使用 DeepSeek API 对样本做五维/六维语义标注，输出：

```text
情感分析/outputs/annotations/
  semantic_annotations_deepseek.jsonl
  semantic_annotations_deepseek.csv
  semantic_annotations_for_review.csv
  semantic_annotations_priority_review.csv
```

最终构建出的可用训练样本为 1349 条，划分为：

| split | 数量 |
|---|---:|
| train | 1079 |
| valid | 135 |
| test | 135 |

训练集标签分布显示，四个主维度存在明显类别不均衡。例如 topic 中 `prepared_food_authenticity`、`meme_offtopic`、`price_value` 较多，而 `food_safety_child_meal`、`industry_trust` 较少；emotion 中 `neutral` 和 `sarcastic` 较多，`angry`、`anxious` 很少。

## 4. 模型选择与训练方式

本项目选择 `hfl/chinese-macbert-base` 作为基座模型。原因是：

1. 它适合中文文本分类任务。
2. 模型规模适中，可以在 MacBook Air M4 的 MPS 后端跑通流程。
3. 后续迁移到 4090 时，可以继续使用同一套脚本，只切换设备后端和 batch size。

训练脚本位于：

```text
情感分析/scripts/finetune/
  train_macbert_single_label.py
  train_macbert_topic.py
  train_macbert_emotion.py
  train_macbert_stance_target.py
  train_macbert_stance.py
```

训练参数保持轻量：

| 参数 | 取值 |
|---|---|
| epochs | 3 |
| batch_size | 4 |
| learning_rate | 2e-5 |
| max_length | 128 |
| weight_decay | 0.01 |
| warmup_ratio | 0.1 |
| seed | 42 |
| 本地设备 | MPS |

本地训练的主要目标不是追求最终性能，而是跑通数据、训练、保存模型、全量预测和分析流程。后续如果迁移到 4090，可以提高 batch size，并加入更系统的超参数搜索和类别不均衡处理。

## 5. 四个分类器结果

四个单标签分类器均已完成本地训练，模型保存在：

```text
情感分析/models/finetuned/
  topic/
  emotion/
  stance_target/
  stance/
```

训练结果如下：

| 分类器 | best valid macro-F1 | test accuracy | test macro-F1 | 主要观察 |
|---|---:|---:|---:|---|
| topic | 0.4335 | 0.5185 | 0.3343 | 大类可用，小类较弱 |
| emotion | 0.2625 | 0.4815 | 0.2237 | neutral/sarcastic 较可用，小情绪弱 |
| stance_target | 0.5196 | 0.6741 | 0.5269 | 四个模型中最稳 |
| stance | 0.4097 | 0.6296 | 0.4014 | question/neutral 较可用 |

这些指标的解释需要谨慎：测试集仍来自 DeepSeek 伪标签，所以它评估的是模型拟合伪标签规则的能力，而不是人工金标准准确率。

## 6. 全量预测

训练完成后，使用四个模型对 40803 条全量评论做预测。预测脚本是：

```text
情感分析/scripts/finetune/predict_semantic_full.py
```

输入：

```text
情感分析/data/raw/semantic_comments_full.csv
```

输出：

```text
情感分析/outputs/predictions/full_semantic_predictions.csv
```

输出文件在原始评论字段之后增加了八列：

```text
pred_topic_label
pred_topic_confidence
pred_emotion_label
pred_emotion_confidence
pred_stance_target_label
pred_stance_target_confidence
pred_stance_label
pred_stance_confidence
```

平均置信度如下：

| 维度 | 平均置信度 |
|---|---:|
| topic | 0.6040 |
| emotion | 0.6697 |
| stance_target | 0.7171 |
| stance | 0.7015 |

这些预测结果成为最终交叉分析的主数据源。

## 7. 训练与预测中的技术收获

### 7.1 MPS 可以跑通流程，但要保守

MacBook Air M4 的 MPS 后端可以完成小规模微调，但需要保守设置：

1. batch size 不宜过大。
2. `num_workers` 设为 0 更稳定。
3. 使用 `PYTORCH_ENABLE_MPS_FALLBACK=1` 允许不支持的算子回退。
4. 推理阶段使用 `torch.inference_mode()`。

### 7.2 4090 迁移应保持脚本参数化

本项目脚本保留了 `--device auto/cuda/mps/cpu` 设计。迁移到 4090 时，主要调整：

1. `--device cuda`
2. 提高 batch size
3. 必要时提高 max_length
4. 使用本地 HuggingFace 模型路径，避免服务器网络问题

### 7.3 类别不均衡是后续优化重点

当前小类标签效果弱，不是模型完全不可用，而是训练数据本身不均衡。后续优化可以考虑：

1. 对小类补充标注样本。
2. 使用 class weight 或 focal loss。
3. 合并过细小类，先做更稳健的中粒度分类。
4. 引入人工复核集，评估真实表现。

## 8. 这一步对项目的意义

模型微调把项目从“抽样分析”推进到“全量语义标注”。有了 `full_semantic_predictions.csv`，我们才能把评论语义与视频源、发布时间、点赞数、回复层级、关系网络等源数据交叉起来，讨论评论区的结构性问题。
