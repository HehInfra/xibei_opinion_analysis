# MacBERT 情绪分析环境与使用说明

本文件说明本项目 MacBERT 情绪分析流程所需的环境、模型与运行方式。

> 本机实际遇到的终端/安装类踩坑记录单独放在本地文件 `踩坑记录.md`（未纳入版本库）。

---

## 1. 环境配置

- **Python**：3.13.6
- **GPU**：NVIDIA GeForce RTX 4060 Laptop（8 GB 显存，约 6.9 GB 可用）
- **已有包**：transformers 4.57.1、scikit-learn 1.7.2、pandas 2.3.1、numpy 2.2.6
- **本流程额外安装**：
  - `tensorflow-cpu 2.21.0`：**仅用于一次性**读取 TF1 checkpoint 完成转换（`load_tf_weights_in_bert` 依赖 TF）。转换结果缓存到 `chinese_macbert_large/hf/` 后，后续训练/预测**不再需要 TF**。
  - `torch 2.9.0+cu126`：GPU 版 torch，启用显卡加速。安装命令：
    ```powershell
    python -m pip install --force-reinstall --no-deps "torch==2.9.0+cu126" --index-url https://download.pytorch.org/whl/cu126
    ```
    验证：`python -c "import torch; print(torch.cuda.is_available(), torch.version.cuda)"`。

---

## 2. 模型说明

`chinese_macbert_large/` 是 **TF1 checkpoint**（`.ckpt.data/.index/.meta`，data 文件约 1.3 GB），不是 HF 格式：

- 架构 = BERT-large（24 层 / hidden 1024 / vocab 21128），即 `hfl/chinese-macbert-large`。
- 需先转换为 PyTorch（脚本 `data/scripts/train_macbert_sentiment.py` 里的 `ensure_pytorch_model()` 已实现，带缓存）。
- `macbert_large_config.json` 没有 `model_type` 字段，但 `BertConfig.from_json_file()` 可正常读取。

> 模型权重文件（`chinese_macbert_large/`、`data/modeling/`）体积较大，不纳入版本库，需本地自行准备。

---

## 3. 运行流程

```powershell
# 前提：已安装 tensorflow-cpu 与 torch+cu126；chinese_macbert_large/ 已就位
$env:TF_CPP_MIN_LOG_LEVEL="3"

# 训练（微调）+ 全量预测
python -u data/scripts/train_macbert_sentiment.py --epochs 4 --batch-size 8

# 常用参数：
#   --convert-only   只做 TF1→PyTorch 转换，不训练
#   --no-train       复用已训练模型，只做预测
#   --limit 200      小规模调试

# 对新评论文件批量推理（复用已微调模型）
python data/scripts/predict_comments_sentiment.py --input <jsonl 或目录> --out <csv>
```

产物在 `data/modeling/`：`macbert_sentiment/`（模型）、`sentiment_predictions.csv`、`sentiment_eval.json`、`summary.md`、`train_run.log`。

情绪标签：`positive / negative / neutral / sarcastic`。

> 运行日志由脚本自身以 UTF-8 写入 `data/modeling/train_run.log`；读取用 `Get-Content <file> -Encoding UTF8` 或编辑器打开，避免乱码。

---

## 4. 性能参考（RTX 4060, batch=8, max_len=128）

- 微调：4 epoch / 455 条样本 ≈ **1.5 分钟**，训练显存峰值 ~4.9 GB。
- 全量预测：40,803 条 ≈ **6 分钟**（~110 条/秒），预测显存 ~1–6 GB。
- `--batch-size 8` 是为了在 8 GB 显存内安全训练（AdamW 优化器状态占显存）。

---

## 5. 结果说明

MacBERT 情绪分类是「规则初标 + DeepSeek 校正 + MacBERT 微调」三步法的第三步。当前训练数据仅 455 条弱标注、类别极不均衡（positive 仅 8 条），所以 neutral/sarcastic 可用（F1≈0.8）、negative/positive 几乎学不到（F1=0）。这是**跑通闭环的原型**，不是高精度模型；提升质量需按方案扩充到 2000+ 条标注再重训。
