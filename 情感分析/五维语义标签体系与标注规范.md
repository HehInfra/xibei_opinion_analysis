# 五维语义标签体系与标注规范

## 1. 文件目的

本文件用于指导西贝预制菜事件 B 站评论的语义标注、模型训练和后续统计分析。

本项目不把评论简单划分为“正面/负面”，而是将每条评论拆解为五个语义维度：

```text
主题：评论在讨论什么
立场：评论支持或质疑谁
情绪：评论表达了什么情感状态
话语方式：评论如何表达观点
风险特征：从信息内容安全视角看，评论包含什么风险表达
```

这五个维度共同服务于项目主题：

```text
面向信息内容安全的争议事件评论多维语义分析：
以 B 站西贝预制菜事件为例
```

## 2. 基本标注原则

### 2.1 以评论文本为主要依据

标注时优先依据评论正文。不要根据标注者个人立场、事件背景偏好或常识推测过多隐藏含义。

如果评论含义不明确，应选择 `other_unclear`、`unclear` 或较低置信度，而不是强行判断。

### 2.2 允许一条评论具有多个语义属性

一条评论可能同时包含：

```text
预制菜主题
质疑西贝立场
讽刺情绪
玩梗话语方式
品牌声誉攻击风险
```

因此，五个维度不是互斥关系。每个维度独立判断。

### 2.3 区分“情绪”和“话语方式”

情绪回答：

```text
评论是什么感受？
```

话语方式回答：

```text
评论是怎么说的？
```

例如：

```text
“一桌菜只有客人是现宰的”
情绪：sarcastic
话语方式：meme, sarcasm
立场：question_xibei
```

### 2.4 区分“立场”和“风险”

质疑西贝不等于风险评论。只有当评论出现明显声誉攻击、未经证实指控、人身攻击、煽动抵制等特征时，才标注相应风险。

例如：

```text
“西贝应该公开说明哪些菜是预制的”
立场：question_xibei
风险特征：low_risk_discussion
```

### 2.5 对反讽和玩梗要结合语境判断

B 站评论中常见反讽、表情、谐音、复读和梗化表达。不要只按字面意思标注。

例如：

```text
“真是太现做了，去年做的也是现做”
字面：肯定
实际：讽刺、质疑
```

## 3. 输出字段设计

建议每条评论输出以下字段：

```text
content_id：评论 ID
content_text：评论文本
topic_label：主题标签
stance_target：立场对象
stance_label：立场方向
emotion_label：情绪标签
discourse_labels：话语方式标签，可多选，用英文分号分隔
risk_labels：风险特征标签，可多选，用英文分号分隔
intensity：表达强度，1-5
confidence：标注置信度，0-1
need_review：是否需要人工复核，true/false
annotation_reason：简短标注理由
```

推荐 JSON 输出格式：

```json
{
  "topic_label": "prepared_food_authenticity",
  "stance_target": "xibei",
  "stance_label": "question",
  "emotion_label": "sarcastic",
  "discourse_labels": ["meme", "sarcasm"],
  "risk_labels": ["reputational_attack", "emotional_amplification"],
  "intensity": 4,
  "confidence": 0.86,
  "need_review": false,
  "annotation_reason": "用玩梗和反讽方式表达对西贝食材真实性的不信任。"
}
```

## 4. 第一维：主题标签

主题表示评论主要讨论什么。原则上每条评论选择一个主主题。

### 4.1 prepared_food_authenticity：预制菜与食材真实性

定义：评论聚焦预制菜、现做、冷冻、隔夜、半成品、食材真实性等问题。

典型表达：

```text
预制菜到底算不算现做
去年羊腿
现宰的只有客人
冷冻半成品
后厨到底有没有现炒
```

边界：

```text
如果评论主要讨论价格，应标 price_value。
如果评论主要讨论知情权，应标 consumer_right_to_know。
```

### 4.2 price_value：价格与性价比

定义：评论聚焦菜品价格、人均消费、贵不贵、值不值。

典型表达：

```text
一个馒头 21 块
人均一百多还用这种料
太贵了
性价比不行
```

### 4.3 food_safety_child_meal：食品安全与儿童餐

定义：评论聚焦食品安全、儿童餐、健康风险、食材保存时间等。

典型表达：

```text
儿童餐
小孩吃这个安全吗
食品安全
过期食材
健康问题
```

### 4.4 brand_pr_crisis：品牌公关与回应翻车

定义：评论聚焦西贝回应、创始人发言、公关策略、危机处理失败。

典型表达：

```text
自杀式公关
越回应越翻车
老板出来越描越黑
公关灾难
```

### 4.5 consumer_right_to_know：消费者知情权

定义：评论聚焦消费者是否有权知道菜品制作方式、是否应明示预制。

典型表达：

```text
消费者应该有知情权
你可以卖预制菜，但要明说
提前告知
别把预制说成现做
```

### 4.6 luoyonghao_persona：罗永浩形象与维权叙事

定义：评论主要讨论罗永浩本人、较真形象、维权经历、直播表现。

典型表达：

```text
老罗还是较真
这个世界需要较真的人
西门子冰箱
罗永浩战斗力
```

### 4.7 industry_trust：餐饮行业信任

定义：评论从西贝扩展到整个餐饮行业、行业规则、连锁餐饮信任问题。

典型表达：

```text
现在餐饮都这样
连锁店是不是都预制
整个行业都该规范
以后不敢在外面吃了
```

### 4.8 meme_offtopic：玩梗或跑题

定义：评论主要是玩梗、接梗、调侃、无明确事件议题。

典型表达：

```text
哈哈哈哈
绷不住了
抽象
客人是现宰的
```

注意：如果玩梗同时明确指向预制菜、价格、公关等议题，应优先标对应实质主题，而不是 `meme_offtopic`。

### 4.9 other_unclear：其他或不明确

定义：无法判断主题、信息不足、纯表情、与事件无关。

## 5. 第二维：立场标签

立场建议拆成两个字段：

```text
stance_target：立场对象
stance_label：立场方向
```

### 5.1 stance_target：立场对象

可选标签：

```text
xibei：西贝或西贝相关负责人
luoyonghao：罗永浩
industry：餐饮行业、预制菜行业或相关监管
consumers：消费者群体或消费者权益
unclear：对象不明确
```

### 5.2 stance_label：立场方向

可选标签：

```text
support：支持、认可、辩护
question：质疑、批评、反对
neutral：中立讨论、解释、补充信息
unclear：无法判断或纯玩梗
```

### 5.3 常见组合

```text
xibei + question：质疑西贝
xibei + support：支持或替西贝辩护
luoyonghao + support：支持罗永浩
luoyonghao + question：质疑罗永浩
industry + question：批评餐饮行业
consumers + support：强调消费者权益
unclear + unclear：纯玩梗或无法判断
```

### 5.4 标注边界

如果评论同时支持罗永浩和质疑西贝，优先标注文本更明确、更主要的对象。若二者强度相近，优先标 `xibei + question`，因为本事件的主要内容安全风险对象是品牌争议。

## 6. 第三维：情绪标签

情绪原则上每条评论选择一个主情绪。

### 6.1 positive：正面或支持

定义：表达赞同、支持、欣赏、鼓励。

典型表达：

```text
支持老罗
这个世界需要较真的人
说得好
```

### 6.2 neutral：中立

定义：客观陈述、解释、补充信息，情绪色彩较弱。

典型表达：

```text
预制菜定义目前比较复杂
这个要看具体加工方式
可以要求明示
```

### 6.3 negative：一般负面

定义：表达不满、批评、失望，但没有明显愤怒或讽刺。

典型表达：

```text
以后不会去了
感觉不太能接受
这事做得不好
```

### 6.4 angry：愤怒

定义：强烈愤怒、责骂、激烈反对。

典型表达：

```text
太离谱了
恶心
坑人
不能忍
```

### 6.5 disappointed：失望

定义：表达失望、信任破裂、遗憾。

典型表达：

```text
以前还挺喜欢西贝的
真的失望
信任没了
```

### 6.6 anxious：担忧

定义：表达食品安全、儿童健康、行业风险方面的担心。

典型表达：

```text
小孩吃这个安全吗
以后外面吃饭都不放心
```

### 6.7 sarcastic：讽刺

定义：用反话、调侃、阴阳怪气表达负面态度。

典型表达：

```text
真是太现做了
客人是现宰的
西贝这波公关太优秀了
```

## 7. 第四维：话语方式标签

话语方式可多选。建议最多选择 3 个主要标签。

### 7.1 rational_argument：理性论证

定义：给出理由、逻辑、因果分析或规范性讨论。

示例：

```text
预制菜不是不能卖，问题是应该提前告知消费者。
```

### 7.2 personal_experience：亲身经历

定义：提到自己或身边人的消费经历、到店经历、感受。

示例：

```text
我之前经常去西贝，价格一直不低。
```

### 7.3 information_claim：事实陈述或爆料

定义：声称某个事实、数据、内幕、菜单、食材、流程。

示例：

```text
他们菜单上这个菜是中央厨房配送的。
```

### 7.4 questioning：质问

定义：用疑问或反问表达质疑。

示例：

```text
这么贵为什么不能现做？
```

### 7.5 sarcasm：反讽或阴阳怪气

定义：表面夸赞或轻描淡写，实际表达批评。

示例：

```text
西贝这公关水平真是遥遥领先。
```

### 7.6 meme：玩梗

定义：使用网络梗、谐音、夸张比喻、接梗、表情包语气等进行表达。

示例：

```text
一桌菜只有客人是现宰的。
```

### 7.7 echoing：附和或复读

定义：简单附和、重复他人观点、接龙。

示例：

```text
确实。
绷不住了。
哈哈哈哈。
```

### 7.8 insult_attack：辱骂攻击

定义：直接辱骂个人、品牌、群体或使用明显攻击性词汇。

示例：

```text
这老板真恶心。
```

### 7.9 simple_attitude：简单表态

定义：只有简短立场或态度，没有明显论证、玩梗或攻击。

示例：

```text
支持老罗。
以后不去了。
```

## 8. 第五维：内容安全风险特征

风险特征可多选。若没有明显风险，标 `low_risk_discussion`。

### 8.1 emotional_amplification：情绪放大

定义：评论通过夸张、强烈情绪、煽动性表达放大不满、愤怒或嘲讽。

示例：

```text
这也太离谱了，谁还敢吃。
```

### 8.2 reputational_attack：品牌声誉攻击

定义：评论直接对品牌信誉、道德、经营行为作负面定性。

示例：

```text
西贝这是把消费者当傻子。
```

### 8.3 unverified_claim：未经证实指控

定义：评论提出缺乏证据的事实性指控、传闻或推断。

示例：

```text
他们肯定很多菜都是放了很久的。
```

注意：如果评论只是表达观点，不提出事实指控，不标此项。

### 8.4 personal_attack：人身攻击

定义：针对个人的人格、外貌、智力、道德进行攻击。

示例：

```text
这个老板脑子有问题。
```

### 8.5 group_mockery：群体嘲讽

定义：嘲讽某类消费者、粉丝、地域、职业或其他群体。

示例：

```text
还去吃的人都是韭菜。
```

### 8.6 boycott_mobilization：抵制动员

定义：明确号召不购买、不消费、举报、抵制。

示例：

```text
大家以后都别去西贝。
```

### 8.7 low_risk_discussion：低风险讨论

定义：普通讨论、温和表达、客观分析，没有明显攻击、煽动或未经证实指控。

示例：

```text
我觉得预制菜可以卖，但应该标清楚。
```

## 9. 表达强度 intensity

表达强度用于衡量评论的情绪和立场强度，取值 1-5。

```text
1：几乎无情绪，仅客观陈述
2：轻微态度或轻微情绪
3：明确态度或中等情绪
4：强烈情绪、明显讽刺、强立场
5：极强情绪、攻击性、煽动性或强烈风险表达
```

示例：

```text
“可以要求明示预制菜” -> 2
“以后不会去了” -> 3
“一桌菜只有客人是现宰的” -> 4
“大家都别去，太恶心了” -> 5
```

## 10. 置信度 confidence 与复核规则

### 10.1 confidence

置信度取值 0-1。

```text
0.90-1.00：含义明确，标签非常确定
0.70-0.89：基本确定，但存在轻微歧义
0.50-0.69：有明显歧义，需要复核
0.00-0.49：无法可靠判断
```

### 10.2 need_review

以下情况建议标记 `need_review = true`：

```text
评论过短，仅有表情或语气词
明显反讽但对象不清
同时涉及多个主题且主次不明
立场对象不明确
风险特征判断存在争议
大模型输出置信度低于 0.70
```

## 12. 面向模型训练的任务设计

由于五个维度性质不同，建议不要训练一个单一五分类模型，而是拆成多个任务。

### 12.1 单标签分类任务

适合：

```text
topic_label
stance_target
stance_label
emotion_label
```

每个字段选择一个主要标签。

### 12.2 多标签分类任务

适合：

```text
discourse_labels
risk_labels
```

一条评论可以同时具有多个话语方式或风险特征。

### 12.3 回归或分级分类任务

适合：

```text
intensity
confidence
```

其中 `intensity` 可以作为 1-5 分类任务，也可以作为有序等级任务。

## 13. 推荐标注流程

```text
1. 从全量评论中抽样 1000-2000 条
   包括随机评论、高赞评论、高回复评论、不同视频和不同时间段评论。

2. 使用本规范进行人工试标
   先标 100-200 条，观察标签是否清晰。

3. 修订标签体系
   合并过细标签，补充高频遗漏标签。

4. 使用大模型辅助标注
   要求大模型按固定 JSON 输出五维标签。

5. 人工抽样复核
   对低置信度、高赞、高回复和风险标签评论重点复核。

6. 形成训练集
   保存为 CSV 或 JSONL，用于微调 MacBERT 或其他中文分类模型。

7. 全量预测
   对 40803 条评论输出五维语义标签。

8. 交叉分析
   将语义标签与视频源、时间、点赞、回复关系和用户信息结合。
```

## 14. 后续分析方向

完成全量标注后，可以围绕以下问题展开：

```text
哪些主题最集中？
质疑西贝是否占据主导立场？
负面表达主要是愤怒还是反讽？
玩梗和反讽是否更容易获得高赞？
高赞评论是否比普通评论更情绪化？
```
