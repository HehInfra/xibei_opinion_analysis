# 四个单标签分类器微调与全量预测总结

本文记录本轮 MacBERT 单标签微调和 40803 条评论全量预测的完整过程，方便后续继续做多标签分类、交叉分析和报告撰写。

## 1. 本轮完成了什么

本轮围绕西贝预制菜事件 B 站评论，完成了四个单标签语义分类器的微调：

```text
topic          主题分类
emotion        情绪分类
stance_target  立场对象分类
stance         立场方向分类
```

并使用这四个模型，对全量评论底表进行预测：

```text
输入：情感分析/data/raw/semantic_comments_full.csv
输出：情感分析/outputs/predictions/full_semantic_predictions.csv
行数：40803
```

## 3. 使用的基座模型

本轮训练使用 Hugging Face 版 MacBERT：

```text
hfl/chinese-macbert-base
```

这个模型已经缓存在本机 Hugging Face cache 中。训练脚本默认从 Hugging Face 名称加载；如果服务器无法联网，可以把模型目录提前下载/拷贝到本地，然后通过：

```bash
--model-name /path/to/chinese-macbert-base
```

显式指定本地模型目录。

注意：

```text
情感分析/models/base/Chinese_macbert_base/ 里已有的本地 MacBERT 是 TensorFlow checkpoint 格式。
本轮没有使用这个 TensorFlow checkpoint，而是使用 Hugging Face PyTorch 格式模型。
```

```text

```

四个任务入口：


四个入口脚本只是传入不同任务名，底层都调用同一个通用训练器，因此训练参数、评估逻辑、模型保存格式是一致的。

## 5. 训练环境与 MPS 问题

使用 conda 环境：

```text
xibei-semantic-ft
```

核心依赖：

```text
torch
transformers
socksio
```

本机训练使用 Apple MPS 后端：

```text
device: mps
```

训练过程中踩过一个重要坑：

```text
在 Codex 默认沙盒内运行 Python 时，PyTorch 会误判 MPS 不可用。
同一个 Python 解释器在沙盒外运行时，MPS 正常可用。
```

因此，后续本机训练和预测要使用 conda 环境中的 Python 直接运行，并显式指定：

```bash
--device mps
```

不要依赖沙盒内的 `conda run ... --device auto`，否则可能退回 CPU。

训练脚本已加入设备诊断，会打印：

```text
device status: {'cuda_available': ..., 'mps_built': ..., 'mps_available': ...}
```

如果本机训练，必须确认：

```text
mps_available: True
```

## 6. 四个分类器如何微调

统一训练参数：

```text
epochs：3
batch_size：4
learning_rate：2e-5
max_length：128
weight_decay：0.01
warmup_ratio：0.1
num_workers：0
device：mps
seed：42
```

训练逻辑：

```text
1. 读取对应任务 CSV
2. 自动根据训练文件中的标签生成 label_to_id / id_to_label
3. 使用 MacBERT tokenizer 对 content_text 分词
4. 加载 AutoModelForSequenceClassification
5. 使用 CrossEntropyLoss 做单标签多分类微调
6. 每轮在 valid split 上计算 loss、accuracy、macro-F1、每类 precision/recall/F1
7. 按 valid macro-F1 保存当前最佳模型
8. 训练结束后在 test split 上评估
9. 保存模型、分词器、标签映射和 training_summary.json
```

模型输出目录：

```text
情感分析/models/finetuned/topic/
情感分析/models/finetuned/emotion/
情感分析/models/finetuned/stance_target/
情感分析/models/finetuned/stance/
```

每个目录中包含：

```text
model.safetensors
config.json
tokenizer.json
tokenizer_config.json
label_mapping.json
training_summary.json
```

## 7. 四个模型训练结果

| 任务 | 设备 | best valid macro-F1 | test accuracy | test macro-F1 |
|---|---:|---:|---:|---:|
| topic | mps | 0.4335 | 0.5185 | 0.3343 |
| emotion | mps | 0.2625 | 0.4815 | 0.2237 |
| stance_target | mps | 0.5196 | 0.6741 | 0.5269 |
| stance | mps | 0.4097 | 0.6296 | 0.4014 |

观察：

```text
1. stance_target 表现最好，test macro-F1 约 0.527。
2. stance 次之，test macro-F1 约 0.401。
3. topic 可用但小类表现不稳，test macro-F1 约 0.334。
4. emotion 最弱，test macro-F1 约 0.224，主要受类别不平衡影响。
```

这些分数不应被解释为人工金标准准确率，只能说明模型对 DeepSeek 伪标签规则的拟合程度。

## 8. 如何做 40803 条全量分类

全量预测脚本：

```text
情感分析/scripts/finetune/predict_semantic_full.py
```

本轮完整预测命令：

```bash
/opt/homebrew/Caskroom/miniconda/base/envs/xibei-semantic-ft/bin/python \
  情感分析/scripts/finetune/predict_semantic_full.py \
  --tasks topic,emotion,stance_target,stance \
  --device mps \
  --batch-size 32 \
  --max-length 128 \
  --output 情感分析/outputs/predictions/full_semantic_predictions.csv
```

预测流程：

```text
1. 读取 semantic_comments_full.csv 中的 40803 条评论
2. 顺序加载四个微调模型
3. 每个模型对 content_text 批量推理
4. 对 logits 做 softmax
5. 取最大概率对应的标签作为预测标签
6. 保存预测标签和置信度
7. 将四个任务预测结果合并回原始评论元数据
```

最终输出：

```text
情感分析/outputs/predictions/full_semantic_predictions.csv
```

文件大小：

```text
约 26M
```

行数：

```text
40803
```

新增字段：

```text
pred_topic_label
pred_topic_confidence
pred_emotion_label
pred_emotion_confidence
pred_stance_target_label
pred_stance_target_confidence
pred_stance_label
pred_stance_confidence
```

## 10. 本轮主要收获

1. 单标签训练流程已经跑通  
   四个任务共用一个通用训练后端，后续扩展和维护成本较低。

2. 本地 MPS 可用于训练和推理  
   但必须在沙盒外运行。沙盒内可能误判 MPS 不可用并退回 CPU。

3. 主题、立场对象、立场方向已经具备初步分析价值  
   其中 stance_target 效果相对最好，适合优先用于后续视频源、时间、点赞和回复网络分析。

4. 情绪分类需要谨慎使用  
   它能较好识别 `neutral`、`sarcastic`、部分 `negative`，但小类如 `angry`、`anxious`、`disappointed` 基本不稳定。

5. 类别不平衡是当前最大限制  
   多个小类在 test 上 F1 为 0 或接近 0。后续如果要提高模型质量，需要补充人工审核或增加伪标签样本。

6. 全量预测结果已经可以进入交叉分析  
   下一步可以按视频源、发布时间、点赞数、回复层级和用户互动关系分析语义分布。

## 11. 注意事项

1. 不要把模型权重提交到 GitHub  
   `情感分析/models/finetuned/` 已被 `.gitignore` 忽略。

2. 不要把当前评估结果写成“人工准确率”  
   训练、验证、测试都来自 DeepSeek 伪标签。

3. 分析时不要只看 accuracy  
   类别不均衡严重，应优先看 macro-F1 和每类 precision/recall/F1。

4. 小类预测要谨慎  
   例如：

```text
food_safety_child_meal
industry_trust
angry
anxious
disappointed
consumers
unclear
```

这些类别要么训练样本少，要么预测数量极少，适合做定性补充，不适合直接作为强结论。

5. 后续如果继续预测，建议显式使用 MPS：

```bash
/opt/homebrew/Caskroom/miniconda/base/envs/xibei-semantic-ft/bin/python \
  情感分析/scripts/finetune/predict_semantic_full.py \
  --device mps
```

6. 如果迁移到 4090，应显式使用 CUDA：

```bash
python 情感分析/scripts/finetune/train_macbert_topic.py --device cuda
```

并先确认：

```bash
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

7. 如果服务器不能访问 Hugging Face，应提前准备本地模型目录，再通过 `--model-name /path/to/model` 加载。
