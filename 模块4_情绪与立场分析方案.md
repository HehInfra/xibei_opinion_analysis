# 模块 4：情绪与立场分析方案

## 1. 模块目标

模块 4 负责回答：

```text
西贝事件相关评论中的情绪倾向是什么？
用户在事件中更倾向于支持谁、质疑谁，还是中立讨论？
```

模块 4 与模块 3 的区别：

```text
模块 3 分析“讨论什么”
模块 4 分析“怎么表达”和“站在哪边”
```

本模块输出将用于：

```text
1. 展示评论区情绪分布
2. 展示评论区立场分布
3. 比较不同视频的情绪与立场差异
4. 支撑模块 6 的评论区圈层化和信息茧房分析
```

## 2. 基本判断

本项目不直接使用通用情感分析包作为唯一判断依据。

原因：

```text
1. B 站评论包含大量反讽、玩梗和口语表达
2. 事件立场和普通情绪不是一回事
3. 通用情感模型可能把讽刺性负面表达误判为中性或正面
4. 课程设计需要可解释标签，而不是纯黑盒分类
```

因此，模块 4 采用：

```text
规则初标 + 事件特定立场体系 + 高赞评论人工校验 + 必要时大模型辅助
```

## 3. 情绪标签体系

情绪标签用于判断评论的语气和情绪方向。

采用单标签策略：

```text
每条评论只保留一个主情绪标签
```

### S1：正面

标签 ID：

```text
positive
```

含义：

```text
表达认可、支持、赞赏、积极评价。
```

典型线索：

```text
支持
认可
真诚
勇敢
需要这样的人
说得对
有道理
```

### S2：负面

标签 ID：

```text
negative
```

含义：

```text
表达不满、愤怒、质疑、失望、批评。
```

典型线索：

```text
离谱
缺德
恶心
失望
太贵
翻车
自爆
割韭菜
不敢吃
```

### S3：中性

标签 ID：

```text
neutral
```

含义：

```text
客观陈述、补充信息、事实讨论，情绪色彩较弱。
```

典型线索：

```text
说明
解释
其实
按照
一般来说
行业里
```

### S4：调侃 / 讽刺

标签 ID：

```text
sarcastic
```

含义：

```text
以玩笑、反讽、夸张、梗化表达为主。通常带有负面或嘲讽倾向，但表达方式不是直接批评。
```

典型线索：

```text
哈哈
笑死
绷不住
doge
大哭
现宰
捅自己
他就先
```

说明：

```text
如果评论既调侃又明显批评西贝，情绪仍可标为 sarcastic，立场再标为 question_xibei。
```

## 4. 立场标签体系

立场标签用于判断评论在西贝事件中的指向。

采用单标签策略：

```text
每条评论只保留一个主立场标签
```

### A1：质疑西贝

标签 ID：

```text
question_xibei
```

含义：

```text
质疑西贝的价格、预制菜、食材真实性、品牌回应、公关处理、消费者权益等。
```

典型线索：

```text
西贝太贵
预制菜
自爆
翻车
公关失败
不真诚
消费者知情权
```

### A2：支持西贝

标签 ID：

```text
support_xibei
```

含义：

```text
替西贝解释、认为西贝被误解、认为预制菜合理、认为争议被夸大。
```

典型线索：

```text
西贝没错
可以理解
行业都这样
没必要骂
预制菜不等于不安全
```

### A3：支持罗永浩

标签 ID：

```text
support_luoyonghao
```

含义：

```text
认可罗永浩较真、维权、揭露问题、推动透明。
```

典型线索：

```text
支持老罗
罗永浩说得对
需要较真的人
为消费者发声
维权
```

### A4：质疑罗永浩

标签 ID：

```text
question_luoyonghao
```

含义：

```text
质疑罗永浩动机、表达方式、是否蹭热度、是否过度攻击。
```

典型线索：

```text
蹭流量
老罗也不是什么好人
炒作
又来了
过度
```

### A5：中立讨论

标签 ID：

```text
neutral_discussion
```

含义：

```text
主要进行事实补充、行业讨论、价格比较、信息说明，不明显支持或反对任何一方。
```

### A6：不明确 / 玩梗

标签 ID：

```text
unclear_meme
```

含义：

```text
文本过短、主要玩梗、上下文缺失，无法可靠判断立场。
```

## 5. 情绪与立场的关系

情绪和立场必须分开判断。

示例：

```text
“一桌菜只有客人是现宰的”
情绪：sarcastic
立场：question_xibei

“这个世界需要一些勇敢的较真的人”
情绪：positive
立场：support_luoyonghao

“预制菜不一定不安全，关键是要告知”
情绪：neutral
立场：neutral_discussion

“罗永浩又来蹭热度了”
情绪：negative
立场：question_luoyonghao
```

## 6. 判定优先级

### 6.1 情绪优先级

当评论同时包含多种情绪时，按以下顺序确定主情绪：

```text
1. sarcastic
2. negative
3. positive
4. neutral
```

原因：

```text
调侃/讽刺在 B 站评论中具有强传播特征，应单独保留；
负面情绪是舆情风险分析重点；
正面和中性按语义强度依次处理。
```

### 6.2 立场优先级

当评论同时涉及多个立场时，按核心攻击或支持对象判断。

默认优先级：

```text
1. question_xibei
2. support_luoyonghao
3. support_xibei
4. question_luoyonghao
5. neutral_discussion
6. unclear_meme
```

原因：

```text
当前事件主要围绕西贝争议展开，质疑西贝是核心舆论方向；
支持罗永浩通常是质疑西贝的伴随立场，但如果评论主要谈罗永浩，则标 support_luoyonghao；
中立讨论和玩梗放在后面。
```

## 7. 分析方法

### 7.1 规则初标

根据关键词、短语和否定表达进行初步标注。

输出：

```text
sentiment_label
sentiment_confidence
sentiment_reason
stance_label
stance_confidence
stance_reason
matched_keywords
need_review
```

置信度规则：

```text
0.90：命中强关键词或明确短语，语义方向清楚
0.75：命中多个普通关键词
0.60：命中少量弱关键词，存在一定歧义
0.40：未命中明确线索或文本过短
```

### 7.2 人工校验

必须人工校验：

```text
1. 高赞评论 Top 50
2. sentiment_confidence < 0.70 的评论
3. stance_confidence < 0.70 的评论
4. sarcastic 评论
5. unclear_meme 评论
6. 同时命中对立立场的评论
```

人工校验字段：

```text
reviewed_sentiment_label
reviewed_stance_label
review_note
reviewer
reviewed_at
```

### 7.3 大模型辅助

如果时间允许，可使用大模型辅助处理低置信度评论。

大模型输入应包含：

```text
评论文本
所属视频标题
模块 3 主题标签
可选：父评论文本
```

大模型输出：

```text
llm_sentiment_label
llm_stance_label
llm_reason
```

大模型结果需要进入人工复核，不直接作为最终唯一依据。

## 9. 质量控制

模块 4 需要记录：

```text
情绪标签分布
立场标签分布
低置信度占比
need_review 占比
高赞评论人工校验完成率
sarcastic 标签占比
unclear_meme 标签占比
```

如果 `unclear_meme` 占比过高，说明规则不足或文本上下文不足。

如果 `question_xibei` 占比极高，需要检查规则是否过度偏向西贝负面关键词。

## 10. 与后续模块的关系

模块 4 输出将直接进入模块 6，用于计算：

```text
情绪一致性
立场一致性
同立场互动比例
跨立场互动比例
高赞评论立场集中度
评论区圈层化指数
```

模块 4 也会辅助模块 5 的网络可视化：

```text
用户节点颜色可以表示主要立场
用户节点边框可以表示主要情绪
回复边可以判断是否跨立场互动
```

## 11. 最终确定方案

本项目模块 4 最终采用：

```text
事件特定标签体系
+ 规则初标
+ 高赞与低置信度人工校验
+ 可选大模型辅助
+ 保留置信度和解释原因
```

不采用：

```text
直接使用通用情感包作为唯一结果
直接训练监督分类模型
直接把大模型输出作为最终结论
```
