# 四维语义标签交叉分析方案

本文档用于指导后续基于 `full_semantic_predictions.csv` 的语义交叉分析。当前先使用已经完成微调和全量预测的四个单标签维度：

```text
topic          主题
emotion        情绪
stance_target  立场对象
stance         立场方向
```

暂不把 `discourse` 和 `risk` 作为主分析维度。原因是这两个任务属于多标签分类，尤其风险标签样本极度不均衡，当前继续微调的边际收益不如先用四维标签完成一轮扎实分析。

## 1. 分析目标

本轮分析要回答的核心问题是：

```text
西贝预制菜事件中，B 站评论区的主题、情绪与立场如何在不同视频、时间、点赞和回复层级中被放大、聚集，并形成局部舆论结构？
```

分析对象不是单条评论，而是大量评论经过语义分类后形成的整体结构。基本思路是：

```text
单条评论语义预测
-> 聚合成整体语义分布
-> 与视频源、时间、点赞、回复层级等源数据交叉
-> 解释争议表达如何被放大、聚集和圈层化
```

## 4. 分析主线一：整体评论语义结构

### 4.1 要回答的问题

```text
评论区整体在讨论什么？
主要情绪形态是什么？
质疑主要指向谁？
哪些主题更容易和反讽、质疑绑定？
```

### 4.2 主要分析表

```text
topic 总分布
emotion 总分布
stance_target 总分布
stance 总分布
topic × emotion
topic × stance
stance_target × stance
```

### 4.3 推荐指标

```text
评论数
评论占比
平均置信度
各类中高点赞评论占比
各类平均 like_count
各类平均 received_reply_count
```

### 4.4 预期可解释现象

```text
1. 事件讨论并不只停留在“预制菜是不是预制菜”，而是扩展到价格、公关、行业信任、消费者知情权。
2. 情绪表达中反讽比直接愤怒更突出，说明争议评论区具有强烈的中文互联网玩梗化表达特征。
3. 质疑主要指向西贝，同时部分评论上升到餐饮行业信任层面。
```

## 5. 分析主线二：不同视频评论区的舆论氛围差异

### 5.1 要回答的问题

```text
不同视频下面的评论区是否形成不同舆论氛围？
哪些视频更偏事实争议？
哪些视频更偏玩梗和反讽？
哪些视频更集中质疑西贝？
```

### 5.2 主要分析表

按 `source_id` / `source_title` 聚合：

```text
视频源 × topic
视频源 × emotion
视频源 × stance_target
视频源 × stance
视频源 × 平均点赞数
视频源 × 平均回复数
```

### 5.3 推荐指标

```text
每个视频评论数
每个视频主题占比
每个视频 sarcastic 占比
每个视频 question 占比
每个视频 stance_target=xibei 占比
每个视频 top topic
每个视频语义集中度
```

语义集中度可以先用简单指标：

```text
最大主题占比 = 某视频中占比最高主题的比例
```

后续也可以计算熵：

```text
主题熵越低，说明该视频评论区越集中。
主题熵越高，说明该视频评论区议题更分散。
```

### 5.4 预期可解释现象

```text
1. 标题强调“品牌翻车”“自爆”的视频，可能更集中于 brand_pr_crisis 和 question。
2. 标题强调“罗永浩”的视频，可能更容易出现 luoyonghao_persona 或支持/质疑罗永浩的讨论。
3. 一些传播量大的视频评论区可能更玩梗化，meme_offtopic 与 sarcastic 占比更高。
```

## 6. 分析主线三：点赞机制下的语义放大

### 6.1 要回答的问题

```text
什么样的评论更容易获得点赞？
高赞评论中的主题、情绪、立场是否更集中？
反讽和玩梗是否比中立表达更容易被顶上去？
质疑西贝是否在高赞评论中更突出？
```

### 6.2 点赞分层建议

可以先按固定阈值分层：

```text
0 赞
1-9 赞
10-99 赞
100-999 赞
1000+ 赞
```

也可以按分位数分层：

```text
top 1%
top 5%
top 10%
其余评论
```

### 6.3 主要分析表

```text
点赞层级 × topic
点赞层级 × emotion
点赞层级 × stance_target
点赞层级 × stance
高赞评论 top topic
高赞评论 top stance_target
高赞评论 top emotion
```

### 6.4 推荐指标

```text
每类平均 like_count
每类中位数 like_count
top 1% 高赞评论中各语义标签占比
top 5% 高赞评论中各语义标签占比
不同语义标签的高赞率
```

高赞率可定义为：

```text
某标签评论中 like_count 进入全体 top 5% 的比例
```

### 6.5 预期可解释现象

```text
1. 玩梗、反讽、品牌翻车类评论可能更容易获得点赞。
2. 高赞评论未必是最理性的信息表达，可能更偏情绪凝练、幽默、讽刺和站队。
3. 点赞机制可能放大某些表达风格，使其成为评论区中更可见的舆论代表。
```

## 7. 分析主线四：评论层级与回复互动

### 7.1 要回答的问题

```text
一级评论和楼中楼回复的语义是否不同？
回复区是否更容易出现争论、附和、反讽和质疑？
高回复评论是否具有更强争议性？
```

### 7.2 主要分析表

按 `content_type` 分组：

```text
comment / reply × topic
comment / reply × emotion
comment / reply × stance_target
comment / reply × stance
comment / reply × 平均 like_count
comment / reply × 平均 received_reply_count
```

再按回复数分层：

```text
received_reply_count = 0
1-9
10-49
50+
```

分析：

```text
回复数层级 × topic
回复数层级 × emotion
回复数层级 × stance
```

### 7.3 推荐指标

```text
一级评论中的主题占比
楼中楼回复中的主题占比
一级评论 question 占比
楼中楼回复 question 占比
高回复评论的 topic / emotion / stance 分布
```

### 7.4 预期可解释现象

```text
1. 一级评论更可能承担观点表达和主题判断功能。
2. 楼中楼回复更可能承担争论、附和、反驳、玩梗接龙功能。
3. 高回复评论可能更集中于争议主题和质疑立场。
```

## 8. 分析主线五：时间演化

### 8.1 要回答的问题

```text
事件讨论是否随时间发生语义迁移？
早期、中期、后期的主题是否不同？
反讽、质疑是否在某些时间段集中爆发？
```

### 8.2 时间粒度

先按日聚合：

```text
date = created_at 的日期
```

如果日粒度不够，再按小时聚合：

```text
hour = created_at 截断到小时
```

### 8.3 主要分析表

```text
日期 × topic
日期 × emotion
日期 × stance_target
日期 × stance
日期 × 评论数
日期 × 平均点赞数
```

### 8.4 推荐图表

```text
每日评论数折线图
每日主题占比堆叠面积图
每日 sarcastic / negative / neutral 占比折线图
每日 question / support / neutral 占比折线图
关键视频发布时间标注
```

### 8.5 预期可解释现象

```text
1. 早期讨论可能集中在预制菜真实性。
2. 随着品牌回应和舆论扩散，讨论可能转向公关危机、价格价值和行业信任。
3. 玩梗和反讽可能出现在事件被广泛传播后的阶段。
```

## 9. 内容安全解释方式

当前暂不直接使用 `risk` 分类器，但仍然可以从内容安全视角解释以下现象：

```text
高反讽 + 高质疑 + 高点赞
-> 情绪化表达的可见性放大

主题高度集中 + 立场高度集中
-> 局部同质化舆论场

楼中楼中 question / sarcastic 聚集
-> 争论、附和和情绪共振

品牌公关危机主题在高赞评论中占比高
-> 声誉风险表达被平台互动机制放大

行业信任主题外溢
-> 单一品牌事件扩展为行业层面信任问题
```

这部分要注意措辞：

```text
不要直接说“违规”或“有害内容”。
更合适的说法是“内容安全风险表达特征”“声誉风险表达”“情绪放大”“立场聚集”“局部圈层化倾向”。
```

## 11. 执行顺序

建议按以下顺序执行：

```text
1. 读取 full_semantic_predictions.csv
2. 做字段类型清洗：created_at、like_count、received_reply_count
3. 生成整体语义分布
4. 生成 topic × emotion、stance_target × stance 等交叉表
5. 生成视频源交叉表
6. 生成点赞分层交叉表
7. 生成评论层级和回复数分层交叉表
8. 生成时间演化表
9. 输出图表
10. 写分析报告
```

## 12. 需要谨慎解释的地方

1. 当前标签来自模型预测，不是人工逐条标注。

2. 模型训练标签来自 DeepSeek 伪标签，因此全量预测继承了 DeepSeek 标注偏好。

3. 小类标签不稳定，例如：

```text
food_safety_child_meal
industry_trust
angry
anxious
disappointed
consumers
unclear
```

4. 情绪模型对小类识别较弱，分析时更适合聚焦：

```text
neutral
sarcastic
negative
positive
```

5. 立场对象中的 `consumers` 数量极少，不宜作为重点结论。

6. 点赞数代表平台内可见性和互动反馈，不等于真实公众态度。

7. 用户公开资料不等于真实人口属性，不应过度做人群画像解释。

## 13. 本轮分析的预期结论方向

如果后续统计结果支持，可以形成以下类型的结论：

```text
1. 评论区核心议题集中在预制菜真实性、玩梗化表达、价格价值和品牌公关危机。

2. 评论区情绪并非单纯愤怒，而是以中性表达和反讽表达为主，体现中文互联网争议事件中的玩梗化舆论特征。

3. 立场上，质疑表达占主导，主要指向西贝，也有一部分讨论外溢到行业信任。

4. 高赞评论可能更偏向反讽、玩梗、品牌危机和质疑表达，说明平台互动机制可能放大更具传播性的情绪化表达。

5. 不同视频评论区可能形成不同舆论氛围，一些视频偏事实争议，一些视频偏玩梗围观，一些视频偏品牌危机叙事。

6. 一级评论和楼中楼回复可能存在表达分工：一级评论承担观点表述，楼中楼回复承担争论、附和和梗化互动。

7. 从内容安全角度看，本事件评论区呈现声誉风险表达、情绪放大和局部立场聚集的特征，但不能简单等同于违规内容。
```
