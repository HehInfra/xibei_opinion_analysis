## 2. 终端 / 操作系统坑（最重要）

### 2.1 一律用 PowerShell，不要用 Bash 工具
本机的 `/usr/bin/bash` 是极简环境，**缺少大量命令**：`ls`、`find`、`head`、`tail`、`cat`(管道)、`sleep`、`git`，而且 **`python` 不在它的 PATH 上**。

最隐蔽的坑：在 bash 里跑 `python xxx.py ; echo EXIT=$?`，即使 python 根本没找到（命令未找到 → 127），结尾的 `echo` 成功会让**整条命令退出码变成 0**，于是任务“成功完成”但脚本其实**一行都没跑**（日志文件不会更新，结果不会变）。我曾因此误判一次。

**结论：所有命令（python、pip、git、nvidia-smi）都用 PowerShell 工具执行。** 文件查找/读取/搜索用专用工具（Glob/Grep/Read），不要用 shell 命令。

### 2.2 PowerShell 跑原生命令的 stderr 会被包装成“报错”
`python`/`pip` 等任何输出到 stderr 的内容（TF 警告、pip notice），在 PowerShell 里会被包成 `NativeCommandError` / `RemoteException`，带一段 `+ CategoryInfo ... + FullyQualifiedErrorId : NativeCommandError`。**这不是真正的失败**，看退出码和实际产物即可。
- 用 `$env:TF_CPP_MIN_LOG_LEVEL="3"` 压制 TF 的 oneDNN/absl 噪声。

### 2.3 日志文件编码：别用 PowerShell 重定向
PowerShell 的 `*>` / `>` 重定向写出的是 **UTF-16 LE（带 BOM）**，会把中文变成乱码、还会破坏后续的 grep/解析。
- **正确做法**：让 Python 脚本自己用 `open(..., encoding="utf-8")` 写日志（本项目脚本已这么做，输出到 `data/modeling/train_run.log`）。
- 读取时用 `Get-Content <file> -Encoding UTF8`，或直接用 Read 工具。

## 3. 安装 / 依赖坑

### 3.1 装 GPU 版 torch：pip 会被“已满足”骗过
直接 `pip install torch==2.9.0 --index-url <cu126>`，pip 认为已安装的 CPU 版 `2.9.0` 已满足要求（裸版本号不区分 `+cpu` / `+cu126`），**什么都不装**。

**正确命令**（用官方索引 + 显式 local version + 强制重装、不动依赖）：
```powershell
python -m pip install --force-reinstall --no-deps "torch==2.9.0+cu126" --index-url https://download.pytorch.org/whl/cu126
```
- 清华 `pytorch-wheels` 镜像目录结构不同，会返回 “versions: none”，**GPU 版 torch 用官方 `download.pytorch.org` 索引**。
- 普通 PyPI 包（tensorflow 等）走清华镜像 `mirrors.tuna.tsinghua.edu.cn` 没问题。
- 官方索引在国内下载约 2.6 GB，可能中途超时，但 pip 会自动断点续传，等它重试即可。
- 验证：`python -c "import torch; print(torch.cuda.is_available(), torch.version.cuda)"`。

### 3.2 safetensors 在 Windows 保存报 os error 1224
`model.save_pretrained()` **覆盖**已存在的 `model.safetensors` 时报
`SafetensorError: ... os error 1224`（`ERROR_USER_MAPPED_FILE`：文件被内存映射占用，很可能是 IDE 索引器/上次加载持有句柄）。
**首次保存成功、覆盖时才失败**是典型症状。

本项目的修复（已写进脚本）：
1. 训练时把最佳权重留在内存，**训练结束只保存一次**（不反复覆盖）。
2. 用 `model.save_pretrained(dir, safe_serialization=False)` → 走 `torch.save` 写 `pytorch_model.bin`，**不走 mmap**。
3. 保存前 `shutil.rmtree(dir)` 清掉旧文件。

## 4. 性能参考（RTX 4060, batch=8, max_len=128）

- 微调：4 epoch / 455 条样本 ≈ **1.5 分钟**，训练显存峰值 ~4.9 GB。
- 全量预测：40,803 条 ≈ **6 分钟**（~110 条/秒），预测显存 ~1–6 GB。
- `--batch-size 8` 是为了在 8 GB 显存内安全训练（AdamW 优化器状态占显存）。

```powershell
$env:TF_CPP_MIN_LOG_LEVEL="3"
python -u data/scripts/train_macbert_sentiment.py --epochs 4 --batch-size 8
# 监视进度：在 IDE 打开 data/modeling/train_run.log（UTF-8）

# 只转换不训练： --convert-only
# 复用已训练模型只预测： --no-train
# 小规模调试： --limit 200
```

## 6. 结果定位提醒

MacBERT 情绪分类是“规则初标 + DeepSeek 校正 + MacBERT 微调”三步法的第三步。当前训练数据仅 455 条弱标注、类别极不均衡（positive 仅 8 条），所以 neutral/sarcastic 可用（F1≈0.8）、negative/positive 几乎学不到（F1=0）。这是**跑通闭环的原型**，不是高精度模型；提升质量需按方案扩充到 2000+ 条标注再重训。
